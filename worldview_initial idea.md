## What this agent should actually be

Call it a **Worldview Agent** or **Meta-Analysis Engine**.

Its job is not to answer questions directly first.
Its first job is to ask:

**“What are all the serious ways this question can be analyzed?”**

So every query should produce six layers:

### 1. Question normalization

Turn vague questions into analyzable ones.

Example:

* User asks: “Will Trump fail by 2027?”
* Agent rewrites:

  * Fail in what sense?
  * Electoral failure?
  * Legislative failure?
  * Economic failure?
  * Approval collapse?
  * Coalition fracture?
  * Institutional blockage?
  * Foreign policy reversal?

This is critical. Most people ask bad questions. Your agent must be ruthless about fixing them.

### 2. Lens expansion

For every question, force a multi-lens breakdown.

Your universal lens library should include:

* Political / institutional
* Geopolitical / military
* Economic / macro
* Financial markets
* Social / demographic
* Media / narrative / propaganda
* Legal / regulatory
* Technological / AI / infrastructure
* Cultural / psychological
* Game theory / incentives
* Historical analogs
* Tail risks / black swans
* Sports / entertainment / domain-specific layer when relevant

This is the heart of the product.

### 3. Scenario generation

For each lens, produce:

* Base case
* Bull case
* Bear case
* Wildcard case

Not one answer. A scenario tree.

### 4. Evidence and indicators

Each scenario must link to:

* supporting evidence
* counter-evidence
* leading indicators
* disconfirming signals

This stops the system from becoming a narrative drug dealer.

### 5. Scoring engine

For each scenario:

* probability
* magnitude of impact
* time horizon
* confidence level
* what would change the score

### 6. Action layer

Depending on the user:

* investor: what positions benefit or get hurt
* operator: what decisions should change
* researcher: what to investigate next
* content creator: what narratives matter
* bettor / forecaster: what markets misprice the scenario

That is where the value lives.

## The main failure modes to avoid

### 1. Endless angle spam

Listing 50 angles is useless if nothing is prioritized.

Fix:

* rank by relevance
* rank by causal importance
* rank by measurability

### 2. Fake balance

Giving symmetrical “for” and “against” takes when one side is much stronger.

Fix:

* require strength-of-evidence scoring

### 3. No definitions

Words like “fail,” “win,” “collapse,” “bubble,” “overvalued” are garbage unless defined.

Fix:

* define success/failure criteria first

### 4. No forecasting discipline

Most agents are essay machines, not forecasters.

Fix:

* force probability ranges
* force time horizon
* force confidence level
* force “what would change my mind”

### 5. No memory

Without stored priors and tracked revisions, you learn nothing.

Fix:

* keep a forecast log
